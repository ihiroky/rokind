# デバッグ用ダミー予定モードの追加

## Summary
- 「ダミーモード」は設定画面内のデバッグオプションとして追加する。
- 起動直後は毎回 `デバッグオプション: 非表示`、`ダミーモード: OFF` を既定値にし、起動をまたいで維持しない。
- ダミーモード中は Google Calendar の自動取得を停止し、手動再取得のみ許可する。
- 予定一覧と通知判定は `Google 手動取得イベント + ローカル追加ダミーイベント` の合成結果を使い、Google とダミーの重なりを競合として表示する。
- 仮想時間は導入せず、時刻基準は常に実時間のままにする。

## Key Changes
- 状態管理
  - `PersistentState` には Google 認証情報、Google イベント、最終同期時刻、エラーのみ保存する。
  - ダミーモード関連は永続化せず、`AppStateStore` にセッション専用状態として持つ:
    - `mock_mode_enabled: bool`
    - `mock_events: Vec<CalendarEventSummary>`
  - アプリ起動時は `mock_mode_enabled = false`、`mock_events = []` で初期化する。
- デバッグオプション UI
  - 設定画面に「デバッグオプションを表示」トグルを追加する。これはフロントエンドローカル状態で持ち、起動時は常に OFF。
  - ダミーモード切替、ダミー予定追加フォーム、ダミー予定削除操作は、このデバッグオプションが ON のときだけ描画する。
  - デバッグオプションが OFF のときは通常運用 UI のみ見せ、ダミー関連操作は表示しない。
- ダミーモードの挙動
  - `mock_mode_enabled = true` の間は 30 秒ポーリングからの Google API 呼び出しをスキップする。
  - `refresh_events` は残し、ダミーモード中でも Google イベントだけを手動更新する。
  - ダミーモードを OFF にしたら `mock_events = []` にリセットし、自動同期を通常どおり再開する。
  - Google 接続解除時は Google 認証情報と Google イベントのみクリアし、ダミーモード中なら `mock_events` は残す。
- イベント合成と競合判定
  - `CalendarEventSummary` に `source: "google" | "mock"` と `has_conflict: boolean` を追加する。
  - 表示用イベント一覧は `google_events + mock_events` を合成して生成する。
  - 競合判定は `[start_at, end_at)` の重複で行い、対象は `google` と `mock` の組み合わせのみ。
  - `end_at` がないイベントは競合判定対象外だが、一覧には表示する。
  - 並び順は開始時刻昇順、同時刻は `google` を先にする。
- 通知
  - ダミーモード中も通知は有効にする。
  - 通知判定は合成イベント一覧を対象にし、既存の実時間基準の Upcoming / StartingNow ロジックをそのまま使う。
  - ダミー予定にも既存のリマインドウィンドウを使い、ソース別の表示分岐は追加しない。

## Public Interfaces
- `src/types.ts`
  - `CalendarEventSummary` に `source`, `has_conflict` を追加。
  - `AppStatus` に `mock_mode_enabled` を追加。
  - `MockEventInput` を追加。
- Tauri commands
  - 追加:
    - `set_mock_mode_enabled(enabled)`
    - `add_mock_event({ title, start_at, end_at, location?, meeting_url? })`
    - `remove_mock_event(event_id)`
  - 既存:
    - `get_app_status` は合成済みイベント一覧を返す。
    - `refresh_events` は常に Google 手動同期として維持する。

## Test Plan
- Rust unit tests
  - 起動初期状態で `mock_mode_enabled = false`、`mock_events = []`。
  - ダミーモード中は自動ポーリングで Google fetch を呼ばない。
  - `refresh_events` はダミーモード中でも Google イベントだけ更新する。
  - ダミーモード OFF で `mock_events` がクリアされる。
  - Google と mock の重なりだけ `has_conflict` が立ち、同一ソース同士では立たない。
- UI/手動確認
  - 起動直後はデバッグUIが非表示で、ダミーモードも OFF。
  - デバッグオプション ON でのみダミー操作UIが見える。
  - ダミー予定追加で一覧に反映され、Google 手動取得イベントとの競合バッジが出る。
  - ダミーモード ON 中でも手動再取得で Google イベントが更新される。
  - アプリ再起動後、デバッグUIは閉じた状態から始まり、ダミーモードとダミー予定は消えている。

## Assumptions
- デバッグオプションの表示ON/OFFはフロントエンドだけの表示状態として扱い、永続化しない。
- ダミーモードOFFへ戻した時点で、検証用状態は残さず毎回クリーンに戻す。
- 時刻基準は常に実時間で、仮想時間の注入や現在時刻オーバーライドは行わない。
